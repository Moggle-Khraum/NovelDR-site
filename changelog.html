Build 1.4.0rev147 Changelog.

v1.4.0 (rev147)

May 7, 2026

✨ "The Foundation Update"

🏗️ Core Storage
  • Migrated entire library storage from AsyncStorage to expo-file-system for better data persistence across app updates.
  • Added automatic data migration on first launch with visual progress tracking.
  • Added multi-location data recovery to find novels after app updates.
  • Added one-time initialization flag so migration only runs once.
📖 Chapter Management
  • Added automatic chapter number extraction from titles to sort chapters correctly regardless of download order.
  • Added chapterNumber field to Chapter type for reliable sorting without repeated title parsing.
  • Enhanced extractChapterNumber to support multiple title patterns (decimal chapters, Roman numerals, etc.).
  • Fixed Table of Contents display where chapters downloaded out of order would appear incorrectly.
  • Migrated to individual chapter file storage for 3,000+ chapters support with 85%+ RAM reduction.
📥 Download & Updates
  • Added chapter number extraction during download for future sorting reliability.
  • Replaced spammy "every 30 chapters" skip logs with cleaner milestone progress at 25%, 50%, and 75% completion.
  • Added final summary log showing total skipped chapters and destination chapter.
  • Fixed chapter sorting to merge existing and new chapters by actual chapter number instead of download sequence.
  • Sorted downloaded chapters by chapter number before saving to library.
  • Added skip progress logging with milestone tracking.
📚 Library Screen
  • Added manual refresh button in header with theme-aware accent color.
  • Added pull-to-refresh functionality to force reload library from storage.
  • Added swipe gesture support for switching between filter tabs.
  • Added search filtering by title or author with batch selection mode.
📖 Reader Screen
  • Fixed gear icon crashing with "Something went wrong" by removing recursive settings calls.
  • Fixed autoscroll not working with smooth continuous scrolling at configurable speeds.
  • Fixed chapter flashing when navigating by persisting previous content during loads.
  • Added AsyncStorage fallback chain for loading legacy chapter content.
  • Fixed TOC chapter sorting with robust number extraction from multiple patterns.
💾 Backup & Restore
  • Added complete backup system saving all novels, chapters, progress, and settings to a single file.
  • Added real-time Activity Log showing each backup and restore step with progress.
  • Added backup file management with share, delete, and metadata preview.
  • Added legacy AsyncStorage data capture in backups for full compatibility.
  • Added Import from File option for external backup restoration.
🎨 UI & Export
  • Added "⋮" menu button to novel detail screen header with export options.
  • Added 5 export formats: TXT, EPUB, DOCX, RTF, and MOBI.
  • Added animated initialization screen with step-by-step progress on first launch.
  • Optimized chapter list with FlatList virtualization for smooth scrolling.
🛠️ Library Context
  • Added refreshLibrary method to manually reload novel data from file system.
  • Updated saveChapterContent to accept optional chapterNumber parameter.
  • Preserved chapterNumber field when saving and loading library metadata.
  • Separated chapter metadata from content for memory-efficient browsing.
🎯 Bug Fixes
  • Fixed issue where chapters without numbers in titles would break sorting.
  • Fixed duplicate chapter entries when updating novels with existing chapters.
  • Fixed data loss when updating from previous app versions.
  • Fixed chapter content not loading from AsyncStorage legacy data.
  • Fixed backup not saving all chapters from multiple novels.
  • Fixed reader settings not persisting after app restart.
  • Fixed progress bar not updating correctly during max chapter limits.
